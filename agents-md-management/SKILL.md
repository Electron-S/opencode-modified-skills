# AGENTS.md 관리

AGENTS.md는 opencode가 프로젝트 컨텍스트로 읽는 파일이다. 대화 맥락에 따라 두 가지 모드로 동작한다.

---

## 모드 1: 세션 학습 기록

**트리거:** 유저가 이번 세션에서 배운 것을 AGENTS.md에 저장하고 싶을 때.

### 1단계: 회고

이번 세션에서 무엇을 발견했나?
- 사용하거나 발견한 빌드/실행/테스트 명령어
- 따른 코드 스타일 패턴
- 효과적이었던 테스트 방법
- 환경/설정 특이사항
- 주의사항이나 함정
- 패키지 관계나 아키텍처 인사이트

### 2단계: 파일 위치 파악

```bash
find . -name "AGENTS.md" -o -name "CLAUDE.md" 2>/dev/null | head -20
```

- `AGENTS.md` — opencode 주 컨텍스트 (git 추적, 팀 공유). **없으면 생성.**
- `CLAUDE.md` — AGENTS.md가 없을 때 폴백

### 3단계: 추가 내용 초안

**간결하게** — 개념당 한 줄. AGENTS.md는 프롬프트의 일부이므로 짧을수록 좋다.

피할 것:
- 장황한 설명
- 코드에서 명백히 알 수 있는 정보
- 재발 가능성 낮은 일회성 수정

### 4단계: 변경 사항 제시

```
### 수정: ./AGENTS.md

**이유:** [한 줄 이유]

```diff
+ [추가 내용]
```
```

### 5단계: 승인 후 적용

유저 승인 후에만 파일을 수정한다.

---

## 모드 2: 품질 감사 및 개선

**트리거:** 유저가 저장소의 AGENTS.md 전체를 점검하거나 개선하고 싶을 때.

### 1단계: 파일 탐색

```bash
find . \( -name "AGENTS.md" -o -name "CLAUDE.md" \) 2>/dev/null | head -50
```

| 파일 | 용도 |
|------|------|
| `./AGENTS.md` | opencode 주 컨텍스트 |
| `./CLAUDE.md` | AGENTS.md 없을 때 폴백 |

### 2단계: 품질 평가

| 기준 | 가중치 | 확인 항목 |
|------|--------|-----------|
| 명령어/워크플로우 문서화 | 높음 | 빌드/테스트/실행 명령어 있는가? |
| 아키텍처 명확성 | 높음 | AI가 코드베이스 구조를 이해할 수 있는가? |
| 비자명 패턴 | 중간 | 함정이나 특이사항이 기록되어 있는가? |
| 간결성 | 중간 | 장황하거나 당연한 정보 없는가? |
| 최신성 | 높음 | 현재 코드베이스 상태를 반영하는가? |
| 실행 가능성 | 높음 | 명령어가 바로 복사해 쓸 수 있는가? |

**등급:**
- **A (90-100)**: 포괄적, 최신, 실행 가능
- **B (70-89)**: 양호, 소소한 공백
- **C (50-69)**: 기본 정보, 주요 섹션 누락
- **D (30-49)**: 부족하거나 오래됨
- **F (0-29)**: 없거나 심각하게 낡음

### 3단계: 품질 리포트

**변경 전 반드시 리포트를 먼저 출력한다.**

```
## AGENTS.md 품질 리포트

### 요약
- 발견된 파일: X개
- 평균 점수: X/100
- 업데이트 필요: X개

### 파일별 평가

#### ./AGENTS.md
**점수: XX/100 (등급: X)**

| 기준 | 점수 | 비고 |
|------|------|------|
| 명령어/워크플로우 | X/20 | ... |
| 아키텍처 명확성 | X/20 | ... |
| 비자명 패턴 | X/15 | ... |
| 간결성 | X/15 | ... |
| 최신성 | X/15 | ... |
| 실행 가능성 | X/15 | ... |

**문제점:** [목록]
**권장 추가 사항:** [목록]
```

### 4단계: 개선안 제시 및 적용

리포트 출력 후 수정 전에 유저 확인을 받는다.

```markdown
### 수정: ./AGENTS.md

**이유:** 빌드 명령어가 없어 혼란 유발.

```diff
+ ## 명령어
+ npm run dev   # 개발 서버 시작 (포트 3000)
```
```

유저 승인 후 Edit 툴로 적용. 기존 구조를 보존한다.

### 자주 발생하는 문제

1. **오래된 명령어** — 더 이상 동작하지 않는 스크립트
2. **환경 설정 누락** — 필요한 환경변수 미문서화
3. **아키텍처 오래됨** — 변경된 파일 구조
4. **미기록 함정** — 비자명 패턴 누락
5. **AGENTS.md 없음** — CLAUDE.md만 존재 (AGENTS.md 생성 권장)
