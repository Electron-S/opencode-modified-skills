## 디자인 아이디어

**지루한 슬라이드를 만들지 않는다.** 흰 배경에 평범한 글머리 기호는 아무도 감동받지 않는다.

### 시작 전

- **내용에 맞는 대담한 색상 팔레트 선택**: 이 주제를 위해 디자인된 느낌이어야 한다. 전혀 다른 프레젠테이션에 색상을 넣어도 "작동"한다면 충분히 구체적인 선택이 아닌 것이다.
- **평등보다 지배**: 한 색상이 시각적 비중의 60-70%를 지배하고, 1-2개의 보조 색상과 하나의 강조 색상. 모든 색상에 동등한 비중을 주지 않는다.
- **명암 대비**: 제목 + 결론 슬라이드에는 어두운 배경, 내용에는 밝은 배경("샌드위치" 구조). 또는 프리미엄 느낌을 위해 전체를 어둡게 유지.
- **시각적 모티프 선택**: 한 가지 독특한 요소를 선택하여 반복 — 둥근 이미지 프레임, 색상 원형 안의 아이콘, 두꺼운 단면 테두리. 모든 슬라이드에 걸쳐 유지.

### 색상 팔레트

주제에 맞는 색상 선택 — 일반적인 파란색을 기본으로 사용하지 않는다:

| 테마 | 주 색상 | 보조 색상 | 강조 색상 |
|------|---------|----------|----------|
| **Midnight Executive** | `1E2761` (네이비) | `CADCFC` (아이스 블루) | `FFFFFF` (흰색) |
| **Forest & Moss** | `2C5F2D` (숲) | `97BC62` (이끼) | `F5F5F5` (크림) |
| **Coral Energy** | `F96167` (산호) | `F9E795` (금색) | `2F3C7E` (네이비) |
| **Warm Terracotta** | `B85042` (테라코타) | `E7E8D1` (모래) | `A7BEAE` (세이지) |
| **Ocean Gradient** | `065A82` (깊은 파랑) | `1C7293` (청록) | `21295C` (자정) |
| **Charcoal Minimal** | `36454F` (차콜) | `F2F2F2` (오프화이트) | `212121` (검정) |
| **Teal Trust** | `028090` (청록) | `00A896` (해초) | `02C39A` (민트) |

### 각 슬라이드에서

**모든 슬라이드에는 시각적 요소가 필요하다** — 이미지, 차트, 아이콘, 또는 도형. 텍스트만 있는 슬라이드는 기억에 남지 않는다.

**레이아웃 옵션:**
- 2단 (텍스트 왼쪽, 그림 오른쪽)
- 아이콘 + 텍스트 행 (색상 원 안의 아이콘, 굵은 헤더, 설명)
- 2×2 또는 2×3 그리드
- 절반 블리드 이미지 (왼쪽 또는 오른쪽 전체) + 내용 오버레이

**데이터 표시:**
- 대형 통계 콜아웃 (60-72pt 큰 숫자 + 작은 레이블)
- 비교 열 (이전/이후, 장단점)
- 타임라인 또는 프로세스 흐름

### 타이포그래피

기본 Arial을 사용하지 않는다. 개성 있는 헤더 폰트와 깔끔한 본문 폰트를 페어링한다:

| 헤더 폰트 | 본문 폰트 |
|-----------|----------|
| Georgia | Calibri |
| Arial Black | Arial |
| Cambria | Calibri |
| Trebuchet MS | Calibri |

| 요소 | 크기 |
|------|------|
| 슬라이드 제목 | 36-44pt 굵음 |
| 섹션 헤더 | 20-24pt 굵음 |
| 본문 텍스트 | 14-16pt |
| 캡션 | 10-12pt 음소거 |

### 간격

- 최소 0.5인치 여백
- 콘텐츠 블록 사이 0.3-0.5인치
- 모든 공간을 채우지 않고 여백을 둔다

### 피해야 할 것 (일반적인 실수)

- **같은 레이아웃을 반복하지 않는다** — 슬라이드에 걸쳐 열, 카드, 콜아웃을 변화
- **본문 텍스트를 가운데 정렬하지 않는다** — 단락과 목록은 왼쪽 정렬; 제목만 가운데
- **크기 대비를 소홀히 하지 않는다** — 14-16pt 본문보다 제목이 돋보이려면 36pt 이상
- **파란색을 기본으로 사용하지 않는다** — 특정 주제를 반영하는 색상 선택
- **간격을 무작위로 혼합하지 않는다** — 0.3인치 또는 0.5인치 간격을 일관되게 사용
- **한 슬라이드만 꾸미고 나머지는 평범하게 두지 않는다** — 완전히 다 하거나 단순하게 통일
- **텍스트만 있는 슬라이드를 만들지 않는다** — 이미지, 아이콘, 차트, 시각 요소 추가
- **텍스트 박스 패딩을 잊지 않는다** — 텍스트 박스에 `margin: 0`을 설정하거나 패딩을 고려하여 도형 위치 조정
- **저대비 요소를 사용하지 않는다** — 아이콘과 텍스트 모두 배경과 강한 대비 필요
- **제목 아래 강조선 절대 금지** — AI 생성 슬라이드의 특징; 여백이나 배경색을 대신 사용

## QA (필수)

**문제가 있다고 가정한다. 찾는 것이 목표다.**

첫 번째 렌더링이 거의 항상 올바르지 않다. QA를 확인 단계가 아닌 버그 헌트로 접근한다.

### 내용 QA

```bash
python -m markitdown output.pptx
```

누락된 내용, 오타, 잘못된 순서 확인.

**템플릿 사용 시 남은 플레이스홀더 텍스트 확인:**

```bash
python -m markitdown output.pptx | grep -iE "xxxx|lorem|ipsum|this.*(page|slide).*layout"
```

grep 결과가 있으면 성공 선언 전에 수정한다.

### 시각적 QA

**⚠️ 서브에이전트 사용** — 2~3개 슬라이드여도 사용한다. 코드를 계속 보다 보면 실제로 있는 것이 아닌 기대하는 것을 보게 된다. 서브에이전트는 새로운 시선을 가진다.

슬라이드를 이미지로 변환(아래 "이미지로 변환" 참고)한 후, 다음 프롬프트를 사용한다:

```
이 슬라이드들을 시각적으로 검사한다. 문제가 있다고 가정하고 찾는다.

확인 사항:
- 겹치는 요소 (도형을 통과하는 텍스트, 단어를 가로지르는 선, 쌓인 요소)
- 텍스트 오버플로 또는 가장자리/박스 경계에서 잘림
- 한 줄 텍스트용으로 배치된 장식선이 두 줄로 줄바꿈됨
- 콘텐츠 위와 충돌하는 출처 인용 또는 바닥글
- 너무 가까운 요소 (0.3인치 미만 간격)
- 불균등한 간격
- 슬라이드 가장자리에서 불충분한 여백 (0.5인치 미만)
- 일관성 없는 정렬
- 저대비 텍스트
- 저대비 아이콘
- 너무 좁은 텍스트 박스로 인한 과도한 줄바꿈
- 남은 플레이스홀더 내용

각 슬라이드에 대해 사소한 것도 포함하여 문제나 우려 사항을 나열한다.
```

### 검증 루프

1. 슬라이드 생성 → 이미지로 변환 → 검사
2. **발견된 문제 목록 작성** (없으면 더 비판적으로 다시 확인)
3. 문제 수정
4. **영향받은 슬라이드 재검증** — 한 수정이 다른 문제를 일으킬 수 있음
5. 전체 통과에서 새 문제가 없을 때까지 반복

**수정 및 검증 사이클을 최소 한 번 완료하기 전에 성공을 선언하지 않는다.**

## 이미지로 변환

시각적 검사를 위해 프레젠테이션을 개별 슬라이드 이미지로 변환한다:

```bash
python ~/.config/opencode/skills/pptx/scripts/office/soffice.py --headless --convert-to pdf output.pptx
pdftoppm -jpeg -r 150 output.pdf slide
```

`slide-01.jpg`, `slide-02.jpg` 등이 생성된다.

수정 후 특정 슬라이드만 재렌더링:

```bash
pdftoppm -jpeg -r 150 -f N -l N output.pdf slide-fixed
```

## 의존성

- `pip install "markitdown[pptx]"` - 텍스트 추출
- `pip install Pillow` - 썸네일 그리드
- `npm install -g pptxgenjs` - 처음부터 생성
- LibreOffice (`soffice`) - PDF 변환 (`scripts/office/soffice.py`로 자동 설정)
- Poppler (`pdftoppm`) - PDF를 이미지로 변환
