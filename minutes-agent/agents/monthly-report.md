# 월간 업무보고 작성 워크플로우

## 워크플로우

1. 대상 연월(YYYY-MM)을 확인한다
2. `output/minutes/파트주간회의/` 에서 해당 월 회의록 파일들을 Read 툴로 모두 읽는다
3. 보고서에 포함할 **업무 주제 목록**을 사용자에게 질문한다
   - 매월 변동 가능 (추가/제거), 사용자가 직접 지정
   - 예시: PMIC 평가 결과 자동 분석 시스템, Display PMIC 평가 자동화 프로그램 개발 등
4. 회의록에 담기지 않은 **추가 업무 내용**을 자유형식으로 받는다 ("없음"도 가능)
5. 회의록 내용 + 추가 업무 내용을 각 주제에 매핑하여 보고서를 작성한다
6. `output/reports/월간보고/YYYY-MM_월간업무보고.md` 로 저장한다 (폴더 없으면 자동 생성)
7. 저장 완료 후 내용을 화면에 출력한다

## 작성 지침

- 회의록 내용을 주제별로 매핑하되, 중복/유사 내용은 통합하여 서술
- 회의록에 없는 내용은 사용자가 제공한 추가 업무 내용으로 보완
- 주제에 매핑할 내용이 없으면 사용자에게 확인 후 생략 또는 "진행 중" 으로 표기
- Confluence에 붙여넣기 가능한 마크다운 형식으로 작성
- **전문 용어는 영어로 표기** (Load Transient, Ripple, Efficiency, EVT/DVT/MP, Vanguard 등)

## 출력 형식 (Confluence 2단계 불릿 구조)

파일명: `YYYY-MM_월간업무보고.md`

```markdown
# YYYY년 MM월 월간 업무보고

## 업무 주제 1
- 첫 1~2문장으로 이번 달 전체 진행 현황 요약.
  이후 구체적인 개발 내용, 완료 항목, 진행 중인 사항, 이슈 및 대응 등을 서술형으로 기술.
  마지막 문장은 업무 효율화·생산성 향상·비용 절감·리스크 감소 등 경영/성과 관점의 가치 문장으로 마무리.

## 업무 주제 2
- ...
```

**작성 규칙**:
- 첫 1~2문장: 해당 주제의 이번 달 전체 현황 요약
- 중간: 구체적 성과, 완료 항목, 진행 중인 사항, 이슈 및 대응 서술
- 마지막 문장: 경영 성과 또는 업무 효율화 관점의 가치 문장
